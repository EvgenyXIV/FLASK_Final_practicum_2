# Flask-Приложение "Личный Дневник"

## Описание проекта

**Личный дневник** — это веб-приложение на основе Flask, позволяющее пользователям создавать, просматривать, редактировать и удалять записи. Проект реализует базу данных с аутентификацией пользователей, сессиями и защитой маршрутов.

---

## Основные функции

- ✅ **Регистрация и вход пользователей**  
  Реализована система регистрации (`/register`) и авторизации (`/login`) с хранением учётных данных в SQLite.
  
- 🔐 **Управление сессиями**  
  Используется токен и временная сессия (30 минут), после чего требуется повторный вход.

- 📝 **CRUD-операции для записей**  
  Пользователи могут:
  - Создавать новые записи (`/form_create`)
  - Просматривать все записи
  - Редактировать существующие по ID
  - Удалять записи

- 🧩 **База данных с миграциями**  
  Используется SQLAlchemy + Flask-Migrate для управления схемой БД (`diary.db`).  
  Модели: `Card` (записи) и `Users` (пользователи).

- 🖼️ **Фронтенд на Jinja2**  
  HTML-шаблоны (`home.html`, `index.html`, `login.html`, `register.html`, `card.html`) с динамическим рендерингом.

---

## Технологии

| Технология       | Назначение |
|------------------|-----------|
| Flask            | Веб-фреймворк |
| SQLAlchemy       | ORM для работы с SQLite |
| Flask-Migrate    | Миграции базы данных |
| Flask-Login      | Управление сессиями пользователей |
| Jinja2           | Шаблонизация |
| Werkzeug         | Хеширование паролей (планируется) |
| SQLite           | Локальная база данных |

> ⚠️ Пароли пока хранятся в открытом виде — для продакшен необходимо добавить `generate_password_hash`.

---

## Структура маршрутов

| Маршрут          | Описание |
|------------------|---------|
| `/`              | Главная страница с операциями над записями |
| `/home`          | Стартовая страница с навигацией |
| `/register`      | Регистрация нового пользователя |
| `/login`         | Вход в систему |
| `/logout`        | Выход из системы |
| `/create`        | Переадресация на форму создания |
| `/form_create`   | Форма создания новой записи |
| `/card`          | Просмотр карточки (защищённый маршрут) |

---

## Особенности реализации

- Глобальные переменные `count` и `change_id` используются для управления состоянием редактирования.
- Обработка ошибок через `try-except` — при ошибках форма перезагружается без падения сервера.
- Все действия доступны только авторизованным пользователям (проверка по токену и времени сессии).

## Запуск проекта

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt

2. Инициализируйте базу данных:
    Bash
    python
    >>> from main import app, db
    >>> app.app_context().push()
    >>> db.create_all()
    >>> exit()

3. Запустите приложение:
    Bash
    python main.py

4. Откройте в браузере: http://127.0.0.1:5000
